# Use cases

## 1. Old battery + new battery
You started with one battery system and later added another one.
Different brand, different parameters, no common logic.

Battery Fusion lets you treat them as one logical storage system.

## 2. DIY battery + commercial ESS
You built your own battery and later added a commercial one (or vice versa).
They don't talk to each other.

Battery Fusion merges them into one operational model inside Home Assistant.

## 3. Inverter shows wrong SOC
Many inverters are not accurate when more than one battery is involved.

Battery Fusion builds its own SOC model based on real energy flow.

## 4. Avoiding expensive EMS hardware
You were told that you need additional hardware to manage multiple batteries.

Battery Fusion shows that in many cases, software + Home Assistant is enough.

## 5. Advanced automation
You want to control:
- when to charge,
- when to discharge,
- how to prioritize energy usage.

Battery Fusion gives you a unified data layer for automation.
